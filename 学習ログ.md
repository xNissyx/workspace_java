# 2026/6/10
実施内容
- 配列

Cとの違い
- 書き方：int[] a = new int[3];
- Javaは配列外参照をゆるさない（コンパイルは通るが、実行時にArrayIndexOutOfBoundsExceptionがでる）

気づき

- Stringはlengthメソッドだが、配列はlengthフィールド

# 2026/6/11
実施内容
- クラス
- コンストラクタ
- メソッド
- フィールド（メンバ変数）
- 修飾子（public/private/static）

気づき
- コンストラクタは名前がクラス名と同じ。戻り値を書かない。new User(...)の時に自動実行される
- コンストラクタを定義してないクラスは自動的にUser()を作成していた！
- 引数の型・数・順番が異なれば別物の扱いになる（オーバーロード）
- private修飾子がついてるものは他のクラスから直接呼ぶことができない（メソッドとか経由すればいける）
- static修飾子は インスタンスではなくクラスに直接属する ようにできる（User.staticMethod()のように呼べる）
- 修飾子の順番は寛容だが、void staticのような戻り値→修飾子の順番にしたらエラー。

# 2026/13
実施内容
- 継承（extends）
- ポリモーフィズム
- インターフェース（※クラスの一種に見えるが概念的には全く異なるもの、型の一種）
- length VS length()
- == VS equals()
- ArrayList
- ジェネリクス

気づき
- 子クラス extends 親クラスで継承できる
- @Overrideでタイポやオーバーライド漏れのコンパイル時チェックが可能になる
- Animal Animal = new Dog(); のように子インスタンスを親型に代入できる
- 呼べるメソッドは変数の型できまるが、実行されるメソッドはオーバーライド時には実体の型の方となる
- インターフェースは「このメソッドを定義してねー」みたいな約束事を書く場所
- インターフェースで定義したメソッドは自動的に public abstract ●● になる。
よって、オーバーライド先でdefault修飾子やprivate修飾子にするとエラーとなる。
- 配列はlengthフィールド、Stringはlengthメソッド
- JavaにおけるリテラルはJVMの文字列専用領域（String Pool）に格納されているので、リテラルは同じ参照となる。
- ArrayList<> 型はimport java.util.ArrayList しないと使えない型。配列と違って動的に要素数を変化可能
- List はインターフェース ArrayListはインターフェースの実装クラス、よって、
List<String> list = new ArrayList<>();はListインターフェース型の実態はArrayListということになる
